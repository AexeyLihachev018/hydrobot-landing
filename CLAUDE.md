# CLAUDE.md — Правила проекта HydroBot

## 🎯 О проекте

HydroBot — Telegram-бот с искусственным интеллектом для производственных компаний в сфере гидравлики (гидроцилиндры, гидростанции, испытательные стенды, насосная техника). Бот обучается на PDF-каталогах и технической документации заказчика, отвечает на вопросы клиентов за 9 секунд, работает 24/7 без участия менеджера. Продукт компании ПромГидравлика-С.

**Задача лендинга:** сконвертировать B2B-трафик в заявку за один визит — через форму или Telegram.

**Сайт:** https://p-gidravlika.ru/

---

## 📁 Структура файлов

- Все страницы — отдельные HTML-файлы (`index.html`, `demo.html`, `privacy.html` и т.д.)
- Стили и скрипты — только внутри HTML-файла (`<style>` и `<script>`)
- Без внешних библиотек, фреймворков и CDN (jQuery, Bootstrap, GSAP и т.д.)
- Без `node_modules`, без сборщиков (Webpack, Vite и т.д.)
- Файл должен открываться двойным кликом — без локального сервера

```
my-landing-pgs/
├── index.html        ← главный лендинг (готов)
├── brief.md          ← стратегический документ
├── research.md       ← анализ конкурентов
├── project.md        ← описание проекта и статус
└── CLAUDE.md         ← этот файл
```

Подключение в `<head>` каждой страницы:
```html

<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
```

- Размер основного текста: 16–18px, `line-height: 1.6`
- Заголовки H1: `clamp(1.9rem, 4.5vw + .4rem, 3.6rem)`, font-weight 800
- Заголовки H2: `clamp(1.55rem, 2.8vw + .4rem, 2.6rem)`, font-weight 700

### UI-паттерны

- Карточки: `border-radius: 10px` (стандарт), `18px` (крупные блоки)
- Кнопки: `border-radius: 10px`, `min-height: 48px`, `transition: all 0.22s ease`
- Секции: `padding: 86px 0` (десктоп), `58px 0` (мобайл)
- Тени: `box-shadow: 0 6px 28px rgba(0,0,0,0.35)`
- Анимации появления: `opacity + translateY` при попадании в viewport через `IntersectionObserver`

## 📱 Адаптивность — обязательно

- **Mobile-first подход** во всех новых файлах
- Breakpoints:
  - `≤ 480px` — мобайл S (iPhone SE и аналоги)
  - `≤ 768px` — мобайл / планшет (основной мобайл)
  - `≤ 1024px` — планшет L
  - `≥ 1280px` — десктоп
- Все tap-targets — минимум **44×44px** (лучше 48×48px)
- `font-size` на `input`/`textarea` — не менее **16px** (иначе iOS масштабирует страницу при фокусе)
- Нет горизонтального скролла ни на одном breakpoint
- Файл открывается из Telegram / WhatsApp без ошибок и без сервера

## ✍️ Стиль текста

- Простой и дружелюбный язык — как разговор с коллегой
- Без канцелярита: не «осуществить», а «сделать»; не «является», а «это»
- Без пассивного залога: не «было создано», а «мы создали»
- Конкретика: цифры, факты, примеры — вместо общих слов («быстро», «удобно», «качественно»)
- Целевая аудитория читает с телефона на производстве — **коротко и по делу**
- Максимум 20 слов в предложении, максимум 3 предложения в абзаце

## 🔄 Рабочий процесс

### Перед созданием нового файла:

1. Напиши что планируешь сделать: название файла, структура, цель страницы
2. Дождись подтверждения от владельца проекта
3. Только потом создавай файл

### Перед изменением существующего файла:

1. Прочитай файл полностью
2. Опиши что именно изменишь и почему
3. Вноси изменения точечно — не переписывай то, что не просили

### При добавлении нового блока на страницу:

- Соблюдай структуру секций из `brief.md`
- Используй существующие CSS-классы (`.btn`, `.tag`, `.sec-head`, `.ai`, `.wrap`)
- Добавляй класс `.ai` на новые элементы для анимации появления
