## 1. ЦЕЛЬ ПРОЕКТА

### 1.1 Бизнес-цель

Снизить нагрузку на технических менеджеров ПромГидравлика-С минимум на 40% за счёт автоматической обработки входящих технических запросов через Telegram-бота HydroBot, обученного на PDF-каталогах и внутренних документах компании.

**Измеримые результаты через 3 месяца:**
- Менеджер тратит на рутинные запросы ≤ 1.5 часа в день (сейчас: 3–4 часа)
- Время первого ответа клиенту ≤ 9 секунд (сейчас: 20–120 минут)
- Конверсия входящих запросов в заявки +15–25%
- Охват нерабочего времени: 100% запросов вне 9:00–18:00 обрабатываются автоматически

### 1.2 Маркетинговая цель лендинга

Сконвертировать входящий трафик (холодный и тёплый) в **квалифицированную заявку** через форму или Telegram в течение одного визита.

**Целевые метрики лендинга:**
| Метрика | Минимум | Цель |
|---------|---------|------|
| Конверсия в заявку | 2% | 4–5% |
| Глубина скролла | 60% | 80% |
| Клик по демо-боту | 15% посетителей | 30% |
| Время на странице | 1:30 мин | 3:00 мин |
| CTR главного CTA | 3% | 7% |

### 1.3 Поведенческая цель пользователя

Посетитель должен пройти путь за один визит:

```
Узнал себя в боли → Понял как работает → Увидел цифры → Поверил → Нажал CTA
     (0–30 сек)        (30–90 сек)        (90–180 сек)   (180–240 сек)  (действие)
```

**Желаемое действие:** нажать кнопку и написать в Telegram или заполнить форму из двух полей: Имя + Telegram/телефон.

## 2. ЦЕЛЕВАЯ АУДИТОРИЯ

### 2.1 Сегменты B2B-аудитории

#### Сегмент A: Собственник / Генеральный директор (решает: да/нет)
- Компания: производство, машиностроение, гидравлика, 20–200 человек
- Боль: видит, что менеджеры загружены рутиной, теряет клиентов на скорости ответа
- Язык: деньги, ROI, конкурентное преимущество, "работает пока сплю"
- Решение принимает: единолично, быстро, если видит ROI

#### Сегмент B: Технический директор / Руководитель отдела продаж (влияет на решение)
- Компания: то же
- Боль: менеджеры тратят время на каталожный поиск вместо продаж
- Язык: эффективность, точность ответов, интеграция в процессы
- Решение принимает: рекомендует собственнику, часто инициирует покупку

#### Сегмент C: Менеджер по снабжению / Технический менеджер (конечный пользователь)
- Боль: 40–60 однотипных вопросов в день, нет времени на сложные задачи
- Язык: "меньше рутины", "не нужно искать в 300-страничном каталоге"
- Решение: не принимает, но его мнение учитывается при внедрении

### 2.2 Боли (дословно, как клиент произносит сам)

1. *"Клиент пишет в 18:30, мы отвечаем на следующий день — он уже купил у конкурентов"*
2. *"Менеджер полдня ищет в каталоге характеристики одного насоса"*
3. *"Новый сотрудник месяц учится, прежде чем нормально отвечать на технические вопросы"*
4. *"Один и тот же вопрос о давлении в системе — я уже наизусть знаю ответ, но каждый раз пишу заново"*
5. *"Нас трое менеджеров, а запросов — как на десятерых"*

### 2.3 Страхи (барьеры к покупке)

| Страх | Что за ним стоит | Как закрыть на лендинге |
|-------|-----------------|------------------------|
| "Бот ответит неправильно — клиент уйдёт" | Репутационный риск | Показать: бот отвечает ТОЛЬКО из ваших документов, неизвестные вопросы эскалирует |
| "Сложно настроить, нужен IT-отдел" | Технофобия, затраты | "Загружаете PDF. Подключаете Telegram. Запускаете. Всё." |
| "Клиенты не хотят общаться с ботом" | Опыт с плохими ботами | Показать реальный диалог с технически грамотными ответами |
| "Дорого и непонятно за что" | Отсутствие ценового ориентира | Цены открыто, расчёт ROI рядом |
| "А вдруг закроетесь через год?" | Надёжность поставщика | Данные о компании, SLA, условия контракта |
| "Команда не примет" | Внутренний саботаж | Отзыв менеджера: "теперь занимаюсь реальными задачами" |

### 2.4 Триггеры принятия решения

**Рациональные:**
- Расчёт: стоимость HydroBot vs. стоимость потерянного рабочего времени менеджеров
- Конкретный кейс из схожей отрасли с цифрами
- Гарантия возврата денег снимает финансовый риск

**Эмоциональные:**
- Страх проигрыша конкурентам, которые уже внедряют AI
- Усталость от рутины — моментальный отклик на боль
- Картина "завтрашнего утра" — менеджер приходит на готовые обработанные запросы

**Логистические:**
- Быстрое внедрение: 1–2 дня, не месяц проекта
- Нет привязки к разработчикам
- Можно попробовать бесплатно до оплаты

## 3. УТП HYDROBOT

### 3.1 Чем отличается от обычного чат-бота

| Обычный чат-бот | HydroBot |
|----------------|---------|
| Отвечает по заранее прописанным сценариям | Отвечает на любой вопрос из ваших реальных документов |
| Требует программирования каждого ответа | Обучается на PDF-каталоге за 1 день без кода |
| Не знает вашу номенклатуру | Знает ваши артикулы, технические характеристики, цены |
| Ломается на нестандартных вопросах | Грамотно эскалирует менеджеру с контекстом |
| Требует постоянной доработки | Обновляется: загрузите новый PDF — бот знает новое |

### 3.2 Почему это не просто "ChatGPT"

ChatGPT не знает:
- Вашу номенклатуру и артикулы
- Ваши реальные цены и сроки поставки
- Ваши технические условия и совместимость
- Внутренние регламенты и условия договоров

HydroBot работает **только с вашими данными**. Он не выдумывает ответы — он цитирует ваши документы. Это критически важно для технических B2B-продаж, где ошибка в характеристике насоса = несостоявшаяся сделка или рекламация.

### 3.3 Конкурентное преимущество для производственной компании

> **HydroBot — это ваша экспертиза, доступная 24/7, для любого клиента, мгновенно.**

Конкретные преимущества для гидравлики/машиностроения:
- Ответ на технический вопрос по каталогу: **9 секунд** вместо 30–90 минут
- Работа в нерабочее время: **100%** запросов обрабатываются без менеджера
- Новый менеджер эффективен с первого дня: бот — его "умная шпаргалка"
- Технические ошибки исключены: бот не путает артикулы и характеристики

### 7.1 Цветовая схема

```
Основной фон:      #0D1117    (глубокий ночной синий — технологичность, серьёзность)
Поверхности:       #161B22    (карточки, панели)
Граница:           #21262D    (разделители, обводки)

Акцентный цвет:    #2563EB    (синий — надёжность, промышленность, технологии)
Акцент hover:      #1D4ED8    (тёмнее при наведении)
Акцент glow:       rgba(37, 99, 235, 0.35)  (свечение кнопок)

Вторичный акцент:  #10B981    (зелёный — "работает", позитивные метрики, "✓")

Текст основной:    #F0F6FC    (почти белый)
Текст вторичный:   #8B949E    (серый, подзаголовки, метки)
Текст мутный:      #484F58    (дата, сноски)
```

**Почему именно эти цвета:**
- Тёмная тема — доминирующий выбор в B2B SaaS 2025-2026 (research.md, Тренд #2)
- Синий (#2563EB) — универсальный цвет доверия в промышленном B2B, ассоциации: надёжность, точность, сила
- НЕ фиолетовый (из research.md): фиолет хорош для потребительских AI-продуктов, но в гидравлике/машиностроении воспринимается как "несерьёзно"
- Зелёный — только для позитивных метрик и состояний "OK"

### 7.2 Шрифты

```
Заголовки H1, H2:    Inter Display (700 Bold) или Geist (800 ExtraBold)
                     Размер: clamp(2rem, 4vw + 0.5rem, 4.5rem)
                     Межстрочный: 1.1

Подзаголовки H3:     Inter (600 SemiBold)
                     Размер: clamp(1.125rem, 2vw, 1.5rem)

Тело (Body):         Inter (400 Regular)
                     Размер: 16-18px, line-height: 1.6

Монospace (код,      JetBrains Mono или Fira Code
артикулы):           Используется в демо-чате Telegram и технических данных

CTA-кнопки:         Inter (600 SemiBold), 16-18px, letter-spacing: 0.01em
```

**Подключение:** Google Fonts — Inter Variable + JetBrains Mono. Preload критических начертаний.

### 7.3 Настроение и ассоциации

**Ключевые слова:** Точность. Надёжность. Скорость. Технологичность без излишества.

**Аналогии из физического мира производства:**
- Как цифровой штангенциркуль: точный, быстрый, профессиональный инструмент
- Как надёжный насос: работает без остановок, не требует постоянного обслуживания
- Как ЧПУ-станок: автоматизирует повторяющиеся операции без потери качества

**Чего избегать:**
- Стоковые фото улыбающихся людей в офисе — сигнал "дешёвый шаблон"
- Яркие поп-цвета, неоновые оттенки — несовместимы с промышленным B2B
- Игривые иллюстрации — для B2C, не B2B
- Тяжёлые 3D-объекты — замедляют загрузку, отвлекают

### Запрещено писать:

| Нельзя | Почему | Заменить на |
|--------|--------|------------|

| "Инновационные технологии AI" | Абстракция без ценности | "Обучается на ваших PDF. Отвечает вашими словами." |
| "Лучший в своём классе" | Недоказуемо | "340+ компаний в гидравлике и машиностроении" |
| "Повышает эффективность" | Без цифры = вода | "Освобождает 3 часа рабочего времени в день" |
| "Нет альтернативы" | Ложь, разрушает доверие | Показать ROI — и сравнение станет очевидным |
| "Работает как магнитная крышка" | Метафора без смысла | Удалено навсегда |
| Технический жаргон без перевода | Отпугивает ЛПР | LLM → "ИИ, обученный на ваших данных" |

### Правила контента:

1. **Каждый абзац** — тест "ну и что?". Если нет конкретной пользы — удалить.
2. **Каждое заявление** — с доказательством: цифра, кейс, или механизм работы.
3. **Каждый страх** закрывается до того, как читатель его сформулирует.
4. **Тон:** не "мы делаем X", а "вы получаете X".
5. **Предложения:** максимум 20 слов. Абзацы: максимум 3 предложения.

```
HTML5 + CSS3 (Custom Properties) + Vanilla JS

Аналитика: Яндекс.Метрика (основная) + Яндекс.Вебвизор
```

### 12.2 Производительность

| Метрика | Цель |
|---------|------|
| PageSpeed Mobile | ≥ 85 |
| PageSpeed Desktop | ≥ 95 |
| LCP (Largest Contentful Paint) | ≤ 2.5 сек |
| CLS (Cumulative Layout Shift) | ≤ 0.1 |
| FID / INP | ≤ 100ms |
| Размер страницы | ≤ 500KB (без изображений) |

**Оптимизации обязательно:**
- Шрифты: `font-display: swap`, preload критических начертаний
- Изображения: WebP с fallback JPEG, `loading="lazy"` для всего ниже fold
- Hero-изображение: `loading="eager"`, максимальное сжатие
- JS: defer для не-критических скриптов, минификация
- CSS: критический CSS inline, остальное async

### 12.3 Адаптивность

| Брейкпоинт | Устройство | Ключевые изменения |

| ≥ 1280px | Desktop | Полная версия |

**Мобайл обязательно:**
- Sticky bottom bar с CTA (появляется после прокрутки 30% страницы)
- Tap target минимум 48×48px
- Accordion FAQ (не hover-раскрытие)
- Touch-friendly carousel для отзывов

- Placeholder: «Опишите проблему или укажите модель оборудования»
- Не обязательное

*Инфо-блок (hint_color, 12px, под формой):*

**Действия:**
- MainButton «Отправить заявку» → валидация → отправка → Экран 5
- BackButton → Экран 3

**MainButton:**
- По умолчанию: «Отправить заявку»
- При отправке: `MainButton.showProgress()` + `MainButton.disable()`
- После успеха: → Экран 5

**Логика отправки (без токена в клиенте):**

## A.5 Чего НЕ будет в первой версии

| Что | Почему не сейчас | Когда |
|-----|-----------------|-------|
| Авторизация / личный кабинет | Лишняя сложность для MVP, 99% пользователей — новые | Этап 3 |
| История заявок | Нужен бэкенд + авторизация | Этап 3 |
| Оплата внутри TMA | B2B не платит онлайн без счёта-фактуры | Никогда (или отдельный этап) |
| Фильтры по параметрам (диаметр/ход) | Сначала нужен контент, потом фильтры | Этап 2 |
| Каталог запасных частей | Сложная логистика, отдельный продукт | Отдельный проект |
| Чат с AI внутри TMA | Требует API-интеграции, бюджета на токены | Этап 4 |
| Push-уведомления об акциях | Нужно разрешение, раздражает | Никогда (только транзакционные) |
| Темная/светлая тема с переключателем | ThemeParams сам подстраивается | Уже решено |
| Анимации и переходы страниц | Усложняет, замедляет старт | По желанию в Этапе 2 |
| Отзывы с рейтингом | Нет верифицированных данных | Этап 2 (с реальными клиентами) |
