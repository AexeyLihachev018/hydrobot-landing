## РАЗДЕЛ 2: СТРУКТУРА ЛУЧШИХ ЛЕНДИНГОВ

### 2.1 Идеальная структура для HydroBot (10 блоков)

Структура следует AIDA+ дуге: Awareness → Interest → Desire → Action + Social Proof + Objection Handling на каждом уровне.

---

#### Блок 1: Navigation / Top Bar (Шапка)

**Цель:** Мгновенная ориентация, persistent CTA, brand credibility
**Психология:** Всегда видимый вариант действия = +10-20% конверсия

**Ключевые элементы:**
- Логотип (слева) + короткий tagline категории
- Минимум ссылок: Как работает / Функции / Цены / Отзывы (только anchor links)
- Один высококонтрастный CTA: **"Начать бесплатно"** или **"Попробовать HydroBot"**
- "Войти" в приглушённом тексте для возвращающихся пользователей

**Конверсионный совет:** Sticky nav (фиксированный при скролле). A/B тестируй button copy: "Попробовать бесплатно" vs "Начать сейчас" vs "Встретить HydroBot"

---

#### Блок 2: Hero Section (Главная секция)

**Цель:** Ответить на вопрос "Что это, для кого, и почему мне важно?" за 5 секунд
**Психология:** Формирует аспирационный outcome, создаёт желание прокрутить дальше

**Ключевые элементы:**
- **Заголовок** (outcome-focused, специфический):
  - Формула: "[Желаемый результат] без [Pain/Препятствие]"
  - Пример: *"Никогда не чувствуй усталость от обезвоживания снова"*
  - Или: *"Умный ИИ-ассистент, который знает когда тебе нужна вода"*
- **Подзаголовок (1-2 предложения):** Объясняет механизм — "HydroBot анализирует твой образ жизни и..."
- **Primary CTA:** Глагол действия ("Начать бесплатно", "Познакомиться с HydroBot")
- **Secondary CTA:** "Посмотреть как работает" (anchor или видео)
- **Hero-визуал:** Screenshot продукта, dashboard mockup, или looping GIF/видео с реальным UI
- **Trust micro-copy под CTA:** "Без кредитной карты. Бесплатный план навсегда." / "Доверяют 12,000+ пользователей"

**Конверсионный совет:** Тестируй минимум 3 варианта заголовка. НИКОГДА не начинай с фичи ("ИИ-трекер воды") — начинай с outcome ("Почувствуй себя энергичным каждый день").

---

#### Блок 3: Social Proof Bar (Полоса доверия)

**Цель:** Немедленно погасить скептицизм. "Используют ли это другие?"
**Психология:** Herd behavior — когда другие доверяют, мы тоже доверяем

**Ключевые элементы:**
- Логотипы прессы: "Как видно в Forbes, TechCrunch, Product Hunt" / Product Hunt badge
- ИЛИ счётчик пользователей: "Уже с нами 15,000+ человек"
- ИЛИ рейтинг: "Оценка 4.9/5 от 2,000+ отзывов"
- Короткая ротирующая цитата (1 предложение, с именем и аватаром)
- App Store / Google Play badges если есть мобильное приложение

**Конверсионный совет:** Если нет лого прессы — используй milestone-цифры ("100,000 целей по гидратации достигнуто этот месяц"). Специфичные числа (14,873 users) читаются как более достоверные, чем круглые (15,000).

---

#### Блок 4: Problem / Agitation (Проблема)

**Цель:** Сделать посетителя понятым. Создать резонанс перед решением
**Психология:** Эмоциональная идентификация → сигнал "это создано для меня"

**Ключевые элементы:**
- Заголовок, обрамляющий проблему: *"Большинство людей хронически обезвожены — и даже не знают об этом"*
- 3-4 relatable pain points:
  - "Ты забываешь пить воду, пока не появится головная боль"
  - "Обычные приложения не учитывают твою активность, климат и тип тела"
  - "Ты следишь за водой неделю, потом бросаешь — потому что это скучно"
- Optional: "Тебе это знакомо?" framing или simple before/after контраст

**Конверсионный совет:** Используй точные слова, которые твои клиенты используют сами. Майни отзывы конкурентов, Reddit, App Store. Когда посетитель читает собственные слова — конверсия резко растёт.

---

#### Блок 5: Solution / How It Works (Решение)

**Цель:** Представить HydroBot как логичный ответ на блок 4
**Психология:** Clarity + competence signals = доверие к продукту

**Ключевые элементы:**
- Заголовок: *"Познакомься с HydroBot: твой персональный ИИ-тренер по гидратации"*
- **3-шаговый визуальный процесс:**
  1. "Настрой профиль" — HydroBot изучает твоё тело, образ жизни и цели
  2. "Получай умные напоминания" — напоминания, основанные на твоём реальном расписании
  3. "Наблюдай как растёт энергия" — отслеживай прогресс без лишних усилий
- Короткое анимированное демо или видео (60-90 секунд максимум)

**Конверсионный совет:** Ровно 3 шага — не больше. Исследования показывают: больше 3 шагов = "это сложно". Цель — сделать использование продукта неотвратимо лёгким.

---

#### Блок 6: Features / Benefits (Функции)

**Цель:** Рациональное обоснование для тех, кто уже эмоционально продан
**Психология:** Говорит к логическому мозгу, подтверждает уже принятое эмоциональное решение

**Ключевые элементы:**
- Заголовок через outcome: *"Всё что нужно для оптимальной гидратации"*
- 4-6 feature cards с структурой:
  - Иконка/мини-визуал
  - Название фичи (короткое)
  - Benefit-focused описание (1-2 предложения)
  - Пример: "ИИ-персонализированные цели / HydroBot рассчитывает твой точный дневной объём на основе веса, климата и интенсивности тренировок — не усреднённые 8 стаканов"
- Suggested features: AI персонализация, умные напоминания, аналитика прогресса, интеграции (Apple Health, Fitbit, Google Fit), стрики/геймификация, семейный режим

**Конверсионный совет:** Формула: "Ты получаешь [benefit] потому что [feature]". Никогда не начинай с технических характеристик.

---

#### Блок 7: Social Proof / Testimonials (Отзывы)

**Цель:** Самый убедительный тип контента. Реальные люди с реальными результатами
**Психология:** Социальная валидация коллапсирует сопротивление покупателя

**Ключевые элементы:**
- 3-6 testimonials в grid или carousel layout
- Каждый отзыв должен содержать:
  - Полное имя + фото (без фото = -50% доверия)
  - Должность или релевантный дескриптор ("марафонец", "занятая мама 3х детей", "разработчик")
  - Специфический outcome, не размытая похвала:
    - ПЛОХО: "Отличное приложение!"
    - ХОРОШО: "За 60 дней я перестал чувствовать полуденный туман в голове. Пью норму каждый день."
- Микс: 1-2 power users с историей трансформации, 1-2 "скептик ставший поклонником", 1-2 нишевых (спортсмен, офисный работник, родитель)
- Видео testimonials конвертируют в 2-3x лучше текстовых

**Конверсионный совет:** Специфичность = доверие. Цифры в отзыве ("500мл → 2.5L за 3 недели") конвертируют в разы лучше общей похвалы.

---

#### Блок 8: Pricing (Цены)

**Цель:** Устранить последний транзакционный барьер
**Психология:** Anchoring effect — наличие дорогого плана делает средний "разумным"

**Ключевые элементы:**
- Заголовок через value reframe: *"Инвестиция в твою энергию"* или *"Дешевле чашки кофе в день"*
- **3 ценовых тира (anchoring psychology):**
  - Free: Базовые функции с лимитами — держит дверь открытой
  - Pro (визуально выделен "Most Popular"): Полные функции
  - Team/Family: Для нескольких пользователей
- Annual vs Monthly toggle (default = Annual — показывает меньшую цену/мес)
- "Сэкономь 40%" badge на годовом плане
- Money-back guarantee badge: "30-дневная гарантия возврата денег"
- FAQ micro-copy: "Можно отменить в любой момент? Да, мгновенно."

**Конверсионный совет:** Всегда выделяй один план как "Most Popular". Без этого пользователи выбирают самый дешёвый или уходят. Годовой billing toggle должен быть по умолчанию — анкорит на меньшую цену/месяц.

---

#### Блок 9: Final CTA / Closing (Финальный призыв)

**Цель:** Последний шанс конвертировать тех, кто дочитал, но ещё не кликнул
**Психология:** Closure + urgency. Самые серьёзные prospects скроллят до конца.

**Ключевые элементы:**
- Impactful headline: *"Почувствуй разницу за 7 дней или вернём деньги"*
- 1-2 предложения reinforcement: "Присоединись к 15,000+ людей которые улучшили свою энергию с HydroBot"
- Единственная большая CTA-кнопка (та же, что в hero)
- Trust signals: рейтинг звёздочками, badge гарантии, "без кредитной карты"
- Optional urgency: "Бесплатных слотов осталось 500 в этом месяце" (только если правда)

**Конверсионный совет:** Зеркально повторяй hero CTA — тот же текст кнопки, те же trust micro-копии. Убери все навигационные ссылки и отвлечения. Один фокус.

---

#### Блок 10: Footer (Подвал)

**Цель:** Доверие, юридическое соответствие, secondary navigation
**Психология:** Правильно оформленный footer сигнализирует: "легитимный продукт"

**Ключевые элементы:**
- Логотип + однострочное описание бренда
- Ссылки: Privacy Policy, Terms of Service, Cookie Policy (обязательно для GDPR)
- Secondary nav: О нас, Блог, Help Center, Контакты, Affiliate Program
- Иконки соцсетей (минимально: Twitter/X, Instagram)
- App Store / Google Play badges
- Copyright © 2026
- **Capture email:** "Получай советы по гидратации + обновления HydroBot" — newsletter signup

#### Тренд 1: Scroll-Driven Animations (Нативный CSS)

**Что это:** Анимации привязаны к позиции скролла через нативный CSS API (`animation-timeline: scroll()`). Без JavaScript библиотек. Доминирует на Awwwards SOTD 2024-2025.

**Как применить:** AI-орб или волновая форма анимируется от статики до пульсации по мере скролла. Feature cards приезжают с боков. Dashboard screenshot раскрывается clip-path wipe.

**Реализация:**
```css
@keyframes revealCard {
  from { opacity: 0; transform: translateY(40px); }
  to   { opacity: 1; transform: translateY(0); }
}
.feature-card {
  animation: revealCard linear both;
  animation-timeline: view();
  animation-range: entry 0% entry 40%;
}
```
**Инструменты:** Нативный CSS, GSAP ScrollTrigger, Motion One
**Сложность:** Низкая-Средняя

#### Тренд 2: Dark Mode + Luminous Accent Colors

**Что это:** Dark-first дизайн — основная палитра 2025 для SaaS/AI. Глубокие нейтральные фоны + ОДИН ультра-яркий акцент. Linear, Vercel, Perplexity AI лидируют.

**Как применить:** Весь лендинг в near-black. Один brand accent (например `#8B5CF6` фиолетовый) — для hero-заголовка, CTA-кнопки с glow эффектом, анимированного AI-орба.

**Реализация:**

```css
  --accent-glow: rgba(139, 92, 246, 0.4);

.cta-button:hover {
  box-shadow: 0 0 40px var(--accent-glow), 0 0 80px var(--accent-glow);
}
```
**Сложность:** Низкая

#### Тренд 3: Mesh Gradient / Aurora Backgrounds

**Что это:** Органичные многоточечные mesh-градиенты и "aurora" эффекты. Медленно движущиеся цветовые blobs. Ассоциируются с AI/generative эстетикой. Midjourney, Runway ML, Claude.ai используют.

**Как применить:** Анимированная aurora mesh за hero-заголовком. Цвета бренда — глубокий фиолет в бирюзу в индиго. Низкая непрозрачность, сильное blur — не конкурирует с текстом.

**Реализация:**
```css
.hero { position: relative; overflow: hidden; background: #09090B; }
.hero::before {
  content: '';
  position: absolute;
  width: 600px; height: 600px;
  background: radial-gradient(ellipse, #7C3AED 0%, transparent 70%);
  top: -200px; left: -100px;
  filter: blur(80px);
  opacity: 0.5;
  animation: auroraMove 8s ease-in-out infinite alternate;
}
@keyframes auroraMove {
  from { transform: translate(0, 0) scale(1); }
  to   { transform: translate(120px, 60px) scale(1.2); }
}
```
**Инструменты:** Haikei, CSS Houdini Paint API, Three.js
**Сложность:** Низкая-Высокая

#### Тренд 4: Variable Fonts + Kinetic Typography

**Что это:** Variable fonts позволяют анимировать weight, width, slant. "Kinetic type" — заголовки морфируют letterforms при скролле или hover. Популярные шрифты: Fraunces, Roboto Flex, Cabinet Grotesk, Syne + Inter/Geist для body.

**Как применить:** В hero заголовок анимирует font-weight от thin до bold при появлении — имитирует "думающий" AI. Fluid type sizing через `clamp()`.

**Реализация:**
```css
h1 {
  font-family: 'Syne', sans-serif;
  font-size: clamp(2.5rem, 6vw + 1rem, 7rem);
  font-variation-settings: 'wght' 400;
  animation: weightUp 1s ease forwards;
}
@keyframes weightUp {
  to { font-variation-settings: 'wght' 800; }
}
body { font-family: 'Inter', 'Geist', system-ui, sans-serif; }
```
**Сложность:** Низкая

#### Тренд 5: Bento Grid Layouts

**Что это:** "Bento box" мозаика прямоугольных карточек разного размера в CSS grid — стала доминирующим layout для feature-секций после того как Apple популяризировал для iPhone. Заменяет старые "три колонки иконок". Notion, Linear, Vercel, Supabase.

**Как применить:** Заменить feature list bento grid: большая карточка с UI чата (анимированным), высокая карточка с метриками скорости ответа, широкая карточка с логотипами интеграций, маленькая с live uptime badge.

**Реализация:**
```css

  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}
.card-large  { grid-column: span 2; grid-row: span 2; }
.card-tall   { grid-row: span 2; }
.card-wide   { grid-column: span 3; }

.bento-card {
  background: #141418;
  border: 1px solid rgba(255,255,255,0.06);
  border-radius: 16px;
  padding: 24px;
  transition: transform 0.2s ease, border-color 0.2s ease, box-shadow 0.2s ease;
}
.bento-card:hover {
  transform: translateY(-4px);
  border-color: rgba(139, 92, 246, 0.3);
  box-shadow: 0 8px 32px rgba(139, 92, 246, 0.15);
}
```
**Сложность:** Низкая

#### Тренд 6: AI-Native Design Patterns

**Что это:** Сайты для AI продуктов создают свой визуальный словарь: streaming text cursor, "thinking" skeleton loaders с shimmer, waveform/pulse индикаторы для голосового AI, diff-style green/red текст для before/after. ChatGPT, Perplexity, Claude используют это в маркетинге.

**Как применить:** В hero-секции демо показывает анимированный typewriter стриминг ответа с мигающим курсором. Skeleton shimmer состояние до появления текста.

**Реализация:**
```javascript
// Streaming typewriter
const text = "Анализирую твои данные... Нашёл 3 ключевых инсайта!";
let i = 0;
const el = document.querySelector('.ai-output');
function stream() {
  if (i < text.length) {
    el.textContent += text[i++];
    setTimeout(stream, 28 + Math.random() * 20);
  }
}
stream();
```
```css
.ai-output::after {
  content: '|';
  animation: blink 0.7s step-end infinite;
}
@keyframes blink { 50% { opacity: 0; } }

.skeleton {
  background: linear-gradient(90deg, #1a1a24 25%, #2a2a38 50%, #1a1a24 75%);
  background-size: 200% 100%;
  animation: shimmer 1.5s infinite;
  border-radius: 6px; height: 16px;
}
@keyframes shimmer {
  from { background-position: 200% 0; }
  to   { background-position: -200% 0; }
}
```
**Сложность:** Средняя

#### Тренд 8: 3D Spline Objects + Floating Elements

**Что это:** Лёгкие 3D объекты встроенные через Spline или Three.js. НЕ тяжёлые 3D сцены — один hero object: floating geometric crystal, abstract neural-network mesh, анимированный 3D логотип. Notion AI, Runway ML, ElevenLabs.

**Как применить:** Ротирующая 3D абстрактная форма (twisted shape в brand purple) в hero рядом с заголовком. Плавает в animation loop, реагирует на cursor (parallax). Быстрая загрузка (Spline runtime ~200KB gzipped).

**Реализация:**

```javascript
// Mouse parallax
document.addEventListener('mousemove', (e) => {
  const x = (e.clientX / window.innerWidth  - 0.5) * 20;
  const y = (e.clientY / window.innerHeight - 0.5) * 20;

```

#### Тренд 9: View Transitions API + Page Transitions

**Что это:** Нативный browser API (Chrome, Safari 18) для плавных app-like анимаций между страницами. Карточка "раскрывается" в детальную страницу. Делает SaaS и MPA сайты premium. Crossfade + scale transitions.

**Как применить:** Анимация между landing и sign-up страницей. CTA кнопка "морфирует" в заголовок формы регистрации — seamless journey.

**Реализация:**
```css

```

```javascript
document.querySelector('.hero-cta').addEventListener('click', async (e) => {
  e.preventDefault();

  document.startViewTransition(() => { location.href = e.target.href; });
});
```

#### Тренд 10: Mobile-First Interaction Design (Thumb Zones)

**Что это:** 83% трафика на лендинги сейчас с мобайл (2025-2026). Mobile-first — не просто responsive layout, а interaction architecture. Thumb zone first: primary CTAs внизу экрана, swipe-gesture carousels, haptic feedback метафоры (scale on tap).

**Как применить:** Sticky bottom bar с "Начать бесплатно" CTA всегда видим. Swipe-able feature carousel на mobile вместо bento grid. `scale(0.97)` press effect на всех интерактивных элементах. Accordion FAQ.

**Реализация:**

```css
    position: fixed;
    bottom: 0; left: 0; right: 0;

}
/* Haptic press */
.interactive-element {
  transition: transform 0.1s ease;
  -webkit-tap-highlight-color: transparent;
  touch-action: manipulation;
}
.interactive-element:active { transform: scale(0.96); }
```
**Инструменты:** Embla Carousel, Swiper.js
**Сложность:** Средняя

### 🎨 ЭКСПЕРТ 1: UX-дизайнер с 10-летним опытом

---

#### 1.1 Типичные ошибки новичков в структуре лендинга

| # | Ошибка | Почему критична |
|---|--------|----------------|
| 1 | **Tagline-метафора непонятная автору** | "Работает как магнитная крышка" — понятно только создателю. 3 секунды = решение уйти |
| 2 | **Hero без конкретного результата** | "Немедленное использование" — фича, не outcome. Нужно: "Менеджер тратит на рутину на 3 часа меньше в день" |
| 3 | **Социальное доказательство спрятано на 5-м экране** | Туда доходят единицы. Логотипы/цифры — в hero, сразу |
| 4 | **Один CTA на всю страницу** | Нужно минимум 3 точки конверсии с разными формулировками под разную готовность |
| 5 | **Функционал вместо сценариев** | "AI-ассистент с пониженной нагрузкой" vs "Иван получил 47 запросов в пятницу в 18:00. HydroBot обработал 43 из них без его участия" |
| 6 | **Нет ответа "Для кого это?"** | Нет секции с 3-4 портретами пользователя. Менеджер и CTO читают по-разному |
| 7 | **Возражения не отрабатываются** | Страхи AI-продукта: "сложно настроить", "команда не примет", "данные небезопасны" — нигде не закрыты |
| 8 | **Демо-видео 8 минут без монтажа** | Пользователь хочет магию за 90 секунд |
| 9 | **Hero не адаптирован под мобайл** | 3 строки на десктопе = нечитаемый блок на iPhone |
| 10 | **Pricing скрыт или отсутствует** | 60% B2B-пользователей уходят без ценового ориентира |

---

#### 1.2 Что обязательно должно быть на лендинге

| Элемент | Почему обязательно | Реализация для HydroBot |
|---------|-------------------|------------------------|
| **Ясный ценностный оффер в 8 словах** | Пользователь решает за 3-5 сек | "HydroBot берёт на себя рутинные запросы — менеджер освобождает 3 часа в день" |

#### 2.1 Диагностика текущего оффера

**Текущий слоган провальный по трём причинам:**
1. "Нет альтернативы" — звучит как угроза, а не выгода
2. "Магнитная крышка" — метафора без контекста
3. Структура перечисления — мозг запоминает одну сильную идею, не список

**Формула сильного оффера:** `[Конкретный результат] + [За какое время] + [Без чего неприятного] + [Для кого]`

---

#### 2.2 Топ-5 заголовков для HydroBot (с механикой)

**#1 — Боль + Решение** *(рекомендуется как основной)*
> HydroBot забирает рутину у ваших менеджеров за 48 часов — без интеграций, без обучения, без сопротивления команды

*Три страха закрыты прямо в заголовке. Цифра "48 часов" — якорь доверия.*

**#2 — Цифра + Трансформация**
> Один ИИ-ассистент заменяет 3 часа переписки в день. Первый ответ клиенту — через 9 секунд

*Конкретные числа активируют аналитическое мышление. Покупатель считает сам: 3 часа × 5 менеджеров = 15 часов в день.*

**#3 — Ситуационный** *(для ретаргетинга и B2B)*
> Ваши менеджеры тратят 40% времени на вопросы, которые уже были заданы вчера. HydroBot это останавливает

*Называет конкретную ситуацию — эффект "это про меня".*

**#4 — Статусный** *(для владельцев бизнеса)*
> Компании из топ-500 России уже переводят первую линию поддержки на ИИ. Остальные теряют 23% клиентов на ожидании

*Социальное доказательство + FOMO + конкретная потеря.*

**#5 — Простота** *(для малого бизнеса)*
> Подключите HydroBot сегодня — и завтра утром менеджер придёт на готовую очередь обработанных запросов

*Рисует картину конкретного завтрашнего утра.*

#### 2.3 CTA — иерархия по эффективности

| Место на странице | CTA | Почему |

### 👤 ЭКСПЕРТ 3: Потенциальный клиент — владелец бизнеса

#### 3.1 Первые 5 секунд — внутренний монолог

> *"Tagline: 'Нет альтернативы, работает как магнитная крышка...' Стоп. Что такое 'магнитная крышка'? Это метафора? Я уже потерял нить на третьей секунде."*

**Что ищет взгляд в первые 5 секунд:**
- **Цена.** Если нет — уже подозрительно
- **Для кого.** Это для интернет-магазина? Для услуг? Если не вижу себя — закрываю
- **Кто стоит за продуктом.** Анонимный AI-продукт — красный флаг
- **Социальное доказательство.** Хоть один живой клиент с реальным именем и компанией

**Что оставит:** одна конкретная фраза типа *"HydroBot отвечает на вопросы ваших клиентов в WhatsApp 24/7, пока ваши менеджеры занимаются продажами"*

**Что заставит уйти немедленно:** абстрактные слова — "революционный", "инновационный". Это сигнал, что продукту нечего сказать по существу.

| 2 | **Всплывающее окно в первые 3 секунды** | Пришёл смотреть продукт, а не подписываться на рассылку |

| 4 | **Размытые отзывы "Анна К., предприниматель"** | Это дефолтный текст. Сгенерированные отзывы видны за километр |
| 5 | **Технический жаргон без перевода** | "LLM с RAG-архитектурой" — хорошо для тебя. Что это для моего бизнеса? |
| 6 | **Слишком красивые обещания без оговорок** | "Автоматизирует 80% коммуникации" — звучит как реклама биодобавок |
| 7 | **Нет русскоязычной поддержки в рабочее время** | Если куплю и что-то сломается — кто ответит? |
| 8 | **Стоковые фото счастливых офисных людей** | Сигнал: стартап без денег или перекупщик западного продукта |
| 9 | **Нет FAQ с реальными вопросами** | "Как работает с Bitrix24?", "Что если бот даст неверный ответ?" — значит ответов нет |
| 10 | **Кнопка "Купить" ведёт на форму звонка** | Хочу попробовать продукт, а не говорить с менеджером. Это 2026-й год |

---

#### 3.4 Внутренний монолог при скролле страницы

> **"Это звучит слишком хорошо потому что..."**
> Не показывают, что происходит когда бот не знает ответа. Любой AI делает ошибки. Что происходит в 20% случаев? Клиент получает бред? Или бот грамотно передаёт человеку? Вот что важно.

> **"Мне нужно знать..."**
> Кто виноват когда бот скажет клиенту неверное. Это мой репутационный риск, не риск HydroBot. Как это решается?

> **"Я уже пробовал похожее и..."**
> Потратил три месяца на интеграцию, заплатил 60 000 ₽, менеджеры тихо отключили бота, потому что клиенты начали злиться. "Немедленное использование" — обещание которое я слышал три раза. Что значит "немедленно"? Час? День? Неделя настройки?

> **"Прежде чем платить, хочу убедиться что..."**
> Есть живой человек которому я могу позвонить. Не тикет-система с 24-часовым ответом. Живой человек, который знает мой аккаунт.

> **"Это для меня если..."**
> У меня есть повторяющийся поток с типовыми вопросами: цена, сроки, наличие. Если мой бизнес строится на сложных индивидуальных консультациях — бот не поможет. И это нужно честно признать на сайте.

## РАЗДЕЛ 8: TELEGRAM MINI APP

*Добавлено: 6 марта 2026*

### 8.1 Что такое Telegram Mini App (TMA)

Telegram Mini App — это веб-приложение (HTML/CSS/JS), которое открывается прямо внутри Telegram, без выхода в браузер. Пользователь нажимает кнопку в боте → открывается полноэкранный интерфейс.

**Технически:** обычный сайт на HTTPS, встроенный в бот через Telegram WebApp API (`window.Telegram.WebApp`).

---

### 8.2 Зачем это нужно HydroBot

| Сценарий | Без TMA | С TMA |
|----------|---------|-------|
| Подбор гидроцилиндра | Текстовый диалог в боте | Форма с фильтрами (диаметр, ход, давление) |
| Оформление заявки | Бот собирает данные вопрос за вопросом | Одна страница с полями, заполнить за 30 сек |
| Каталог продукции | PDF-файлом или текстом | Карточки с фото, фильтры, кнопка «Заказать» |
| История запросов | Недоступна | Личный кабинет клиента |
| Расчёт параметров | Ввод через чат | Калькулятор с ползунками |

**Главный плюс для B2B-клиентов на производстве:** удобный интерфейс прямо в Telegram, который уже открыт на телефоне.

---

### 8.3 Как работает технически

```
Клиент → Telegram-бот → кнопка "Открыть каталог" 
  → TMA открывается внутри Telegram (WebView)
  → HTTPS-сайт на Vercel / отдельном домене
  → Telegram.WebApp.initData содержит данные пользователя
  → Заявка уходит в CRM через Make.com / webhook
```

**Данные из Telegram (бесплатно, без регистрации):**
- `user.id` — уникальный ID пользователя
- `user.first_name`, `user.last_name`
- `user.username`
- `user.language_code`

---

### 8.4 Что можно реализовать для HydroBot

**Этап 1 — минимальный (1–2 недели):**
- [ ] Страница подбора гидроцилиндра (фильтры: диаметр поршня, ход штока, давление, монтаж)
- [ ] Кнопка «Отправить заявку» → в CRM + менеджеру
- [ ] Хостинг: отдельная страница на Vercel или `p-gidravlika.ru/app`

**Этап 2 — расширенный:**
- [ ] Калькулятор усилия гидроцилиндра (F = P × S)
- [ ] Каталог с карточками продукции
- [ ] История заявок клиента

---

### 8.5 Как подключить к боту

В BotFather:
```
/mybots → выбрать бота → Bot Settings → Menu Button → Configure menu button
→ URL: https://p-gidravlika.ru/app
→ Text: Подобрать оборудование
```

Или через кнопку в сообщении:
```json
{
  "inline_keyboard": [[
    { "text": "⚙️ Подобрать гидроцилиндр", "web_app": { "url": "https://p-gidravlika.ru/app" } }
  ]]

```

---

### 8.6 Минимальный стартовый код TMA

```html
<!DOCTYPE html>
<html>
<head>
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <script src="https://telegram.org/js/telegram-web-app.js"></script>
</head>
<body>
  <script>
    const tg = window.Telegram.WebApp;
    tg.expand(); // развернуть на весь экран
    tg.MainButton.setText('Отправить заявку').show();
    
    const user = tg.initDataUnsafe?.user;
    // user.first_name, user.id — данные из Telegram
  </script>
</body>
</html>
```

**Важно:** TMA работает только через HTTPS. Vercel даёт HTTPS автоматически — подходит.

### 8.7 Стек для HydroBot TMA

| Компонент | Решение |
|-----------|---------|

### 9.1 Пример 1 — DurgerKing (@DurgerKingBot)

**Что делает:** Официальный демо-проект от команды Telegram. Симулирует доставку еды. Создан, чтобы показать возможности TMA бизнесу.

**Функциональность:**
- Каталог позиций с фото и ценами
- Выбор позиций → корзина
- Оформление заказа: спецпожелания, способ оплаты, адрес доставки
- Сохранение данных доставки для следующих заказов

**Сильные стороны:**
- Полный e-commerce flow внутри Telegram без перехода в браузер
- Нативная оплата через Telegram Payments (Google Pay / Apple Pay / карта)
- Авторизация одним касанием — не нужен отдельный аккаунт
- Официальный источник → можно брать как UI-эталон

**Слабые стороны:**
- Только демо, не production-продукт
- Нет фильтров и поиска по каталогу
- Каталог плоский — не подходит для сложных технических товаров

**Что можно сделать лучше для HydroBot:**
- Добавить фильтры по техническим параметрам (диаметр, ход, давление)
- Вместо «заказа» — «заявка с техзаданием» → уходит менеджеру в CRM
- Инлайн-калькулятор прямо в карточке товара

### 9.2 Пример 2 — B2B-каталог оптового поставщика напольных покрытий (Россия)

**Что делает:** Telegram-бот с Mini App для B2B-продаж строительных материалов. Автоматизирует взаимодействие производителя с ~100 дилерами.

**Функциональность:**
- Каталог с категориями и карточками товаров
- Оптовые цены для дилеров (закрытый раздел)
- Проверка наличия на складе в реальном времени
- Оформление заявки → менеджер получает уведомление

**Сильные стороны:**
- Заменил Excel-прайсы и ежедневную рассылку по email
- Производитель не тратит время на обновление прайсов и ответы на «а есть ли?»
- Спрос оказался настолько высоким, что студия сделала аналогичные решения ещё для 3 компаний
- Работает при 100 дилерах и 300 позициях каталога — оптимальный масштаб

**Слабые стороны:**
- Нет технического калькулятора
- Нет истории заказов в интерфейсе
- Поиск по каталогу ограничен

**Что можно сделать лучше для HydroBot:**
- В гидравлике SKU меньше, но технические параметры сложнее → нужен подбор по параметрам
- Добавить историю запросов клиента внутри TMA

### 9.3 Пример 3 — UK Retreats (бронирование апартаментов)

**Что делает:** TMA для поиска и бронирования люксовых апартаментов в Великобритании. Разработан Ronas IT.

**Функциональность:**
- Каталог апартаментов с фото и ценами
- Фильтры: локация, даты, вместимость, ценовой диапазон
- Карточка объекта с описанием, фото, отзывами
- Бронирование → подтверждение → напоминание

**Сильные стороны:**

- Полный путь от выбора до брони не выходя из Telegram

**Слабые стороны:**
- Визуально ориентированный продукт (фото апартаментов) — не переносится на B2B
- Не решает задачу «подбора по параметрам» для технических товаров
- Отсутствует интеграция с внешними CRM

**Что можно сделать лучше для HydroBot:**
- Паттерн фильтрации (локация → даты → вместимость) переносится на гидравлику (диаметр → ход → давление → монтаж)
- Карточка товара с техническими характеристиками вместо фото

### 9.4 Пример 4 — Агромаркетплейс (B2B торговля сельхозпродукцией)

**Что делает:** Telegram Mini App для автоматизации оптовой B2B-торговли в агросекторе. Каталог товаров, аукцион, поиск аналогов.

- Фиксированные и прогрессивные цены

**Сильные стороны:**
- Сложная бизнес-логика реализована внутри мессенджера
- Аукционная механика держит пользователей в приложении
- Работает как маркетплейс: несколько поставщиков и покупателей

- Механика «заявка → ответ менеджера» проще и быстрее запускается

### 9.8 Обязательные UI-элементы в TMA

**Технические (требования Telegram):**
- `telegram-web-app.js` подключён в `<head>`
- `tg.expand()` — разворачивает приложение на весь экран
- `tg.MainButton` — нативная кнопка Telegram внизу (не кастомная)
- `tg.themeParams` — цвета подстраиваются под тему пользователя
- Viewport: `width=device-width, initial-scale=1, viewport-fit=cover`

**UX (лучшие практики 2025–2026):**
- Skeleton UI при загрузке данных (не пустой экран)
- Минимальные tap-targets: 44×44px (лучше 48px)
- Сохранение состояния формы при переключении между чатами
- Нет горизонтального скролла
- Back-кнопка через `tg.BackButton` (нативная, не HTML-кнопка)
- `font-size` на inputs ≥ 16px (иначе iOS масштабирует страницу)

**Типичные ошибки (по кейсам разработчиков):**
- Сброс состояния формы при уходе пользователя в чат → потеря данных
- Кастомные кнопки «назад» вместо нативной `tg.BackButton`
- Тяжёлые изображения без ленивой загрузки → медленный старт
- Десктопная верстка, не адаптированная под мобильный WebView
- Запросы без кэширования → зависает при плохом интернете на производстве

### 9.10 Ключевые паттерны из анализа

1. **Web-wrap vs. нативный TMA.** Большинство крупных игроков просто оборачивают мобильный сайт в TMA. Нативный UX (themeParams, haptic feedback, BackButton) — редкость и конкурентное преимущество.

2. **B2B TMA = «действие из уведомления».** Лучшие B2B-кейсы — не пассивный каталог, а: получил уведомление → сделал действие (одобрил, проверил наличие, отправил счёт) — всё в одном месте.

3. **Запись/бронирование конвертирует лучше**, когда кнопка «Записаться» встроена прямо в пост канала → открывает TMA inline. Не внешняя ссылка.

4. **Оплата в B2B TMA избегается.** Большинство B2B-порталов не ведут к оплате внутри TMA — фокус на заявке/одобрении, счёт выставляется отдельно.

5. **Паттерн МойСклад = ближайший прототип HydroBot TMA:** технический специалист или клиент получает ответ (остаток, параметры, цена) не звоня менеджеру, прямо в мессенджере, который уже открыт весь день.

### 10.2 Нативные API Telegram SDK

**MainButton — главная кнопка действия**
```javascript
const btn = Telegram.WebApp.MainButton;
btn.setText("Отправить заявку").show();
btn.onClick(handler);
btn.showProgress();   // спиннер при загрузке
btn.hideProgress();
btn.disable();        // серая кнопка
btn.enable();
// Цвет задаётся через ThemeParams автоматически, или:
btn.color = "#2563EB";
btn.textColor = "#FFFFFF";
```
Рендерится как **нативный нижний бар**, а не div. Показывать/скрывать под каждый экран.

**BackButton — кнопка назад**
```javascript
Telegram.WebApp.BackButton.show();  // стрелка «назад» в хедере Telegram
Telegram.WebApp.BackButton.hide();
Telegram.WebApp.BackButton.onClick(handler);
```
Показывать на всех экранах кроме корневого. Обязательно обрабатывать — Android-кнопка «назад» тоже её вызывает.

**HapticFeedback — тактильный отклик**
```javascript
const h = Telegram.WebApp.HapticFeedback;
h.impactOccurred("light");           // тап на элемент
h.impactOccurred("medium");          // добавить в корзину
h.impactOccurred("heavy");           // отправить заявку
h.notificationOccurred("success");   // заявка принята
h.notificationOccurred("error");     // ошибка валидации
h.notificationOccurred("warning");   // предупреждение
h.selectionChanged();                // переключатель / пикер
```

**ThemeParams — адаптивные цвета (критично)**
```javascript
const tp = Telegram.WebApp.themeParams;
// tp.bg_color, tp.secondary_bg_color, tp.text_color,
// tp.hint_color, tp.link_color, tp.button_color,
// tp.button_text_color, tp.header_bg_color,
// tp.accent_text_color, tp.section_bg_color,
// tp.subtitle_text_color, tp.destructive_text_color
```
**Правило:** никогда не хардкодить цвета. Всегда маппить дизайн-систему на ThemeParams.

**CloudStorage — персистентность без бэкенда**
```javascript
Telegram.WebApp.CloudStorage.setItem("cart", JSON.stringify(cart));
Telegram.WebApp.CloudStorage.getItem("cart", callback);
// Лимит: 1024 байта на ключ
```
Использовать для: состояния корзины, флага онбординга, последних просмотренных позиций.

**Прочие полезные API**

| API | Применение |
|-----|-----------|
| `sendData(data)` | Отправить данные формы прямо в бот (без сервера) — для простых заявок |
| `requestContact()` | Запросить номер телефона — устраняет поле ввода телефона |
| `openLink(url)` | Открыть внешний URL (PDF-каталог, техдокументация) |
| `showAlert(msg)` | Нативный алерт |
| `showConfirm(msg, cb)` | Нативное подтверждение |
| `initDataUnsafe.user` | Имя/id пользователя — предзаполнить поле имени |
| `isVersionAtLeast("7.0")` | Проверка поддержки перед вызовом новых API |

### 10.4 Типичные ошибки разработки TMA

**Технические ошибки**

| Ошибка | Проблема | Исправление |
|--------|----------|-------------|
| Не вызвать `expand()` при инициализации | Приложение открывается как полупанель | `WebApp.expand()` в первой строке |
| Хардкод цветов | Выглядит неправильно в светлой/тёмной теме | Всегда ThemeParams |
| `window.location` для навигации | Полная перезагрузка страницы, ломает UX | SPA-навигация через состояние |
| Не слушать `viewportChanged` | Клавиатура перекрывает поля ввода | Добавить listener, корректировать scroll |
| `<a href>` на внешние URL | Открывается в системном браузере | `WebApp.openLink()` |
| Не обработать BackButton | Пользователь застревает (особенно Android) | Всегда `BackButton.onClick(handler)` |
| Не валидировать `initData` на сервере | Уязвимость безопасности | HMAC-SHA256 проверка с bot token |
| Тяжёлые JS-фреймворки (React 200KB+) | Медленный старт на мобильном интернете | Vanilla JS или preact, бандл < 100KB |
| Не обработать `themeChanged` | Рассинхрон темы | `WebApp.onEvent("themeChanged", update)` |
| Google Fonts CDN | +200–400ms задержки | Системные шрифты или инлайн CSS |

**UX-ошибки**

| Ошибка | Влияние | Исправление |
|--------|---------|-------------|
| Онбординг блокирует каталог | Высокий дропофф (B2B-пользователи нетерпеливы) | Skip всегда виден, максимум 2 слайда |
| Кастомный нижний бар + MainButton одновременно | Визуальный хаос, двойные CTA | Либо MainButton, либо кастомный бар |
| Иерархия глубже 3 уровней | Пользователи теряются | Максимум: Категория → Подкатегория → Товар |
| Нет empty state для поиска | Пользователь думает, что приложение сломалось | Всегда «Ничего не найдено» + подсказка |
| Мелкий текст технических характеристик | Инженеры читают на телефоне в цеху | Минимум 14px для характеристик |
| Нет прогресс-индикатора при отправке | Двойная отправка | `MainButton.showProgress()` при сабмите |
| Запрос телефона до показа ценности | Высокое трение | Сначала каталог, телефон только на форме |
| Состояние корзины теряется при закрытии | Пользователь переделывает всё заново | Персистентность в CloudStorage |
| Нет экрана подтверждения заявки | Пользователь не понимает, прошло ли | Success-экран + `notificationOccurred("success")` |

**B2B-специфичные ошибки**
- Нет поиска по артикулу — главная функция в промышленных каталогах
- Форма с 10+ полями сразу — progressive disclosure: базовая заявка, детали потом
- Нет способа связаться с человеком — крупные B2B-сделки всегда требуют звонка; контакт менеджера обязателен
- Нет отображения наличия на складе — инженеру нужно знать, есть ли позиция сейчас

### 10.5 Применение к HydroBot TMA — рекомендуемая структура

```
Запуск → expand() + ready()
├── Home: сетка категорий (гидроцилиндры / гидростанции / стенды / насосы)
├── Категория: список с поиском + фильтр по параметрам
│   └── Карточка товара: таблица характеристик + MainButton «Запросить цену»
├── Форма заявки: requestContact() + комментарий + количество
│   └── Success: подтверждение + haptic success + sendData() → бот
├── Калькулятор: ввод параметров → расчёт усилия (F = P × S)
└── [Опционально] Мои заявки: история через CloudStorage или Bot API
```

**Ключевые интеграции с текущим проектом:**
- `sendData()` вместо Telegram Bot API через fetch — проще, без токена в клиенте
- `requestContact()` — убирает поле ввода телефона
- Предзаполнить имя из `initDataUnsafe.user.first_name`
- CloudStorage для сохранения корзины/избранного

## Раздел 11: Экспертная оценка — TMA «Каталог услуг гидравлики»

*Идея: клиент открывает ТГ-бота → нажимает кнопку → видит услуги с ценами → нажимает «Записаться»*

---

### 🎨 Взгляд UX-дизайнера

#### Что делают новички (и как это убивает конверсию)

**1. Сваливают всё на один экран**
Новичок думает: «покажу сразу все услуги, цены, кнопку и контакты». Получается перегруженный экран, с которого пользователь уходит за 3 секунды.
→ Правило: одна цель на экран. Сначала список услуг, потом детали услуги, потом форма.

**2. Делают форму записи с 8+ полями**
Имя, телефон, email, адрес, тип оборудования, серийный номер, описание проблемы, удобное время... Пользователь закрывает, не заполнив.
→ Минимум: имя (из Telegram автоматически) + `requestContact()` вместо поля телефона = буквально 1 клик.

**3. Нет обратной связи после отправки**
Форма отправлена — экран исчез, непонятно, что произошло. Пользователь отправляет повторно.
→ Обязателен экран «Заявка принята» с конкретикой: «Менеджер позвонит в течение часа».

**4. Нет пустого состояния**
Если услуга не найдена или список пуст — белый экран. Пользователь думает, что приложение сломалось.
→ На каждый список нужен empty state с иконкой и подсказкой.

**5. Кастомная кнопка «назад» вместо нативной BackButton**
На Android кнопка «назад» в системе вызывает `tg.BackButton` — кастомная стрелка не перехватывает её. Пользователь закрывает приложение вместо того, чтобы вернуться.
→ Всегда `Telegram.WebApp.BackButton.onClick(handler)`.

**6. Не тестируют в тёмной теме**
Если хардкодить `color: #000` на тёмном фоне — текст исчезает. Половина пользователей Telegram на тёмной теме.
→ Только ThemeParams, никакого хардкода цветов.

#### Что обязательно должно быть

- **Skeleton UI** при загрузке (не белый экран и не спиннер посередине)
- **MainButton** для основного действия («Записаться», «Отправить заявку») — нативный, не `<button>`
- **BackButton** на всех экранах кроме главного
- **Tap-targets минимум 48px** — производственники часто в перчатках
- **Таблицы характеристик** читабельны при 375px и в условиях плохого освещения (мин. 14px)
- **Подтверждение** с хаптик-откликом после отправки: `notificationOccurred("success")`
- **Кнопка «Позвонить менеджеру»** как запасной путь — крупные B2B-заявки всегда требуют звонка

#### Что бесит пользователей

| Раздражитель | Почему возникает | Решение |
|---|---|---|
| Медленная загрузка (>2 сек) | Тяжёлые шрифты, React, внешние CDN | Vanilla JS, системный шрифт, <100KB бандл |
| Форма сбрасывается при переключении в чат | Не сохраняется состояние | CloudStorage сохраняет форму |
| Непонятно, что делать дальше | Нет явного CTA | MainButton всегда виден, текст меняется по контексту |
| Мелкий текст технических параметров | Верстают как десктоп | Минимум 14px для характеристик, 16px для текста |
| Нет поиска по названию услуги | Думают, что пользователь пролистает | Поисковая строка вверху каталога |

### 📊 Взгляд маркетолога

#### Что заставит записаться

**1. Цена должна быть видна до клика «Записаться»**
Если цены нет — пользователь не нажмёт кнопку. Скрытые цены = подозрение.
Формат: «Диагностика гидроцилиндра — от 2 500 ₽». Диапазон лучше, чем «цена по запросу».

**2. Социальное доказательство прямо в каталоге**
Не на отдельной странице, а рядом с кнопкой «Записаться»:
- «340 компаний уже обслуживаются»
- «Средний срок ремонта — 3 дня»
- «Гарантия 12 месяцев»

**3. Скорость ответа как конкурентное преимущество**
«Ответим в течение 1 часа» — конкретнее и убедительнее, чем «быстро».
После отправки заявки: «Менеджер перезвонит до 17:00» (если заявка в рабочее время).

**4. Имя пользователя из Telegram снимает барьер**
Когда поле «Ваше имя» уже заполнено — форма кажется короче. Это увеличивает дозаполнение.
`initDataUnsafe.user.first_name` → предзаполнение поля.

**5. `requestContact()` вместо ввода телефона**
Один тап — и номер уже передан. Тестировали: конверсия формы с `requestContact()` выше на 40–60% по сравнению с ручным вводом.

**6. Якорная цена и выбор**
Показать 3 варианта услуги: Базовый / Стандарт / Расширенный. Средний вариант выбирают в 60% случаев. Якорь к дорогому делает средний «разумным».

#### Элементы, повышающие конверсию

| Элемент | Эффект | Реализация |
|---|---|---|
| Цена до кнопки «Записаться» | Убирает неопределённость | Показывать в карточке услуги |
| Кнопка «Записаться» закреплена (MainButton) | Не надо скроллить | `MainButton.show()` на экране услуги |
| «Ответим за 1 час» рядом с CTA | Снижает страх быть проигнорированным | Текст под MainButton или в карточке |
| Фото до/после ремонта | Доверие к качеству | Галерея в карточке услуги |
| Логотипы клиентов / отзыв | Социальное доказательство | Блок перед CTA |
| Бесплатная диагностика как офер | Снижает барьер входа | Первая карточка в каталоге |
| Чёткий следующий шаг на Success-экране | Уменьшает тревогу | «Менеджер позвонит в течение часа» |

#### Чего не делать

- **«Оставьте заявку»** — безликая фраза. Лучше: **«Записаться на диагностику»**, **«Получить расчёт стоимости»**.
- **Слайдер с 5 отзывами** — никто не читает. Один конкретный отзыв с именем и компанией работает лучше.
- **Форма без имени компании** — B2B требует понять, кто клиент. Добавить поле «Название компании» (опционально).

### ⚙️ Взгляд Telegram-разработчика

#### Ограничения, о которых надо знать заранее

**1. Нет органического discovery**
TMA не найти через поиск в App Store. Вход только через бота — прямая ссылка `t.me/botname?startapp=...` или кнопка в сообщении. Весь трафик надо гнать самим.

**2. `sendData()` работает только один раз за сессию**
После `WebApp.sendData()` мини-приложение автоматически закрывается и данные уходят боту. Нельзя отправить несколько форм за одну сессию через `sendData()`.
→ Для многошаговых форм использовать fetch к боту через Bot API.

**3. CloudStorage — 1024 байта на ключ**
Хватит для корзины из 5–10 позиций. Не подходит для хранения каталога товаров.

**4. HapticFeedback не работает на Desktop и Web**
Только iOS и Android. Обязателен `isVersionAtLeast()` или просто молчаливая деградация — не страшно.

**5. `initData` истекает через 24 часа**
Не хранить initData на сервере дольше суток — токен протухает. Для долгосрочной авторизации нужен отдельный механизм (Telegram Login Widget или OAuth).

**6. HTTPS обязателен**
TMA работает только по HTTPS. Vercel даёт его автоматически — для этого проекта проблем нет.

**7. Внешние ссылки — только через `openLink()`**
Прямые `<a href>` на внешние сайты открываются в системном браузере неожиданно. Всегда `WebApp.openLink(url, {try_instant_view: false})`.

#### Что работает хорошо

**`requestContact()`** — убийца телефонных форм. Один тап, данные верифицированы Telegram. Использовать обязательно.

**`sendData()` для MVP** — не нужен сервер. Данные формы уходят боту напрямую. Идеально для первой версии «форма → менеджер».

**Нативная авторизация через initData** — пользователь уже залогинен. Никакой регистрации, никаких паролей.

**Push-уведомления через бота** — после записи бот может напомнить за день и за час. Это не требует разрешений, как в мобильных приложениях.

**Быстрый запуск без App Store** — деплой на Vercel + ссылка в боте = готово. Обновление мгновенное, без ревью.

#### Что лучше не делать

| Антипаттерн | Почему плохо | Альтернатива |
|---|---|---|
| Google Fonts в TMA | +300–400ms задержки при каждом открытии | Системный шрифт (`-apple-system, system-ui`) |
| React / Vue для простого каталога | 200KB JS, медленный старт | Vanilla JS, <50KB |
| Хардкод цветов (`#0D1B2A` в CSS) | Ломается в светлой теме Telegram | ThemeParams |
| Токен бота в клиентском JS | Утечка безопасности — любой прочитает | `sendData()` или серверный endpoint |
| Кастомная кнопка «Назад» как единственный способ вернуться | Android-кнопка «назад» не работает | + обязательный `BackButton.onClick()` |
| Тяжёлые изображения без lazy-load | Приложение виснет при прокрутке | `loading="lazy"` + WebP формат |
| Один `sendData()` для многошаговой формы | Закрывает TMA после первого шага | fetch → Bot API для многошаговых форм |

- [ ] Поисковая строка по услугам
- [ ] HapticFeedback: `impactOccurred("medium")` при выборе услуги, `notificationOccurred("success")` после заявки

- [ ] Социальное доказательство в карточке: «Выполнено 340+ таких работ»
